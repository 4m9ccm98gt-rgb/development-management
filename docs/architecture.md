# 全体構成

## development-management

開発の司令塔です。プロジェクト横断の現在地、ルール、ロードマップ、設計判断、配布、障害対応、AI引き継ぎを管理します。コード本体と実運用設定は持ちません。

## next-day-setup

翌日準備業務のアプリケーションです。PMS等の情報を取り込み、Google Sheetsのシフトを参照し、席割・担当割・各種帳票・印刷を扱います。印刷プラットフォームと共有版配布物はこのプロジェクトが担当します。

## inventory-reconciliation-system

販売在庫チェック専用システムです。PMS、手間いらず、JTBの在庫情報を照合し、差異の出力、夜間自動実行、警告メールを担当します。休館日はGoogle Sheetsを正式参照元とし、取得失敗時はキャッシュを利用します。印刷機能は持ちません。

## 関係

- 2つの業務システムは別リポジトリ、別`.venv`で独立して運用する。
- シフト／休館日の参照元としてGoogle Sheetsを共有するが、用途と処理は各プロジェクトが担当する。
- 印刷責務は `next-day-setup` に集約する。
- 開発状況と判断は `development-management` に集約し、詳細実装は各業務リポジトリに残す。
