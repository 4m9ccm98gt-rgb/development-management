# トラブルシューティング

## 共有版だけ起動しない

- 開発環境と共有版のファイル差、Pythonランタイム、`_internal` の残存ファイルを確認する。
- `dist\DinnerSystem` 単体で起動できるか確認し、専用スクリプトで共有版を再同期する。
- 実運用設定や業務データは削除しない。

## `numpy.short` 等のNumPyエラー

- 異なるPython／NumPy版のファイルが `_internal` に混在していないか確認する。
- 正式な`.venv`でクリーンビルドし、`_internal` を完全同期する。

## 古い `_internal` が残る

- `robocopy /E` はコピー元にない古いファイルを削除しない。
- `_internal` に限定して完全同期する専用スクリプトを使う。共有フォルダ全体へ `/MIR` は使わない。

## PowerShell文字コード

- Windows PowerShell 5.1で解析できる構文か確認する。
- 日本語を含む場合は実行環境と互換性のある文字コードで保存し、実機で起動確認する。

## `master_settings.json` 不在によるビルド失敗

- 実運用ファイルをGitへ追加しない。
- `master_settings.example.json` とビルド処理の期待値を確認し、秘密情報を含めずにビルドできる設計か確認する。
- 必要な実運用設定は配布先で保持する。

## 正式ソースと旧フォルダの混同

- 作業前に現在パスを確認する。
- `Development\repos` 配下以外は参照専用。修正は正式ソースに反映して検証する。

## Codexサンドボックス権限制約

- 正式ソース外への書き込み、Git設定、ネットワーク操作が制限される場合がある。
- 権限エラーをコード不具合と混同しない。対象パスと許可範囲を確認し、必要な操作だけ承認を得る。
- 回避のために旧フォルダや別コピーを新たな開発元にしない。
