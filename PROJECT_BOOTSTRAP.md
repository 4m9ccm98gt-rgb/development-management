# Project Bootstrap

この文書は、単体タスクや試作を、AIと継続開発できる正式プロジェクトへ昇格させる標準手順です。既存資産を保護し、正式ソース、GitHub、文書、`development-management` の状態を一致させて進めます。

## プロジェクト化チェックリスト

### 1. 現状調査と計画

- [ ] 対象の目的、利用者、現行運用、関連資産を確認する。
- [ ] 既存ファイル、未コミット変更、実運用データ、機密情報の有無を確認する。
- [ ] 移行対象、移行しない対象、検証方法、切り戻し方法を整理する。

### 2. GitHubリポジトリと正式ソース

- [ ] GitHubリポジトリを作成し、所有者、公開範囲、既定ブランチを確認する。
- [ ] 正式ソースのローカルパスを決定する。
- [ ] 旧フォルダや試作場所との役割を分け、以後の変更先を正式ソースへ統一する。
- [ ] GitHubから取得した状態を確認し、初回同期方法を決定する。

### 3. プロジェクト文書

- [ ] `README.md` に目的、対象範囲、構成、起動方法、開発方法、確認方法を記載する。
- [ ] `docs/` を整備し、設計、仕様、運用、トラブルシューティングなど継続開発に必要な文書を配置する。
- [ ] `AI_HANDOFF.md` または同等のAI引き継ぎ文書を作成し、正式ソース、現状、禁止事項、未完了作業、確認方法を記載する。
- [ ] `.gitignore` を整備し、生成物、ローカル設定、実行時データ、秘密情報を除外する。

### 4. 安全性とGit管理対象

- [ ] APIキー、パスワード、トークン、認証情報が含まれていないことを確認する。
- [ ] 顧客データ、業務データ、個人情報、実運用設定がGit管理対象に含まれていないことを確認する。
- [ ] サンプル設定と実値を分離し、必要な設定手順を文書化する。
- [ ] 追跡対象と除外対象を確認し、不要な大容量ファイルや生成物がないことを確認する。

### 5. development-management登録

- [ ] `projects/<project-name>.md` を作成し、目的、正式ソース、現状、次の作業、未確認事項を記録する。
- [ ] [PROJECT_STATUS.md](PROJECT_STATUS.md) にプロジェクトの現在地と次の作業を追加する。
- [ ] [VERSION_MATRIX.md](VERSION_MATRIX.md) にGitHub、開発版、実運用版、デモ版などの対応状況を追加する。
- [ ] [REPOSITORIES.md](REPOSITORIES.md) にGitHubリポジトリと正式ローカルパスを追加する。
- [ ] [CHANGELOG.md](CHANGELOG.md) にプロジェクト化と確認状況を記録する。

### 6. 初回Git同期

- [ ] 正式ソースでブランチ、remote、差分を確認する。
- [ ] README、docs、AI引き継ぎ、`.gitignore`、ソースの内容をレビューする。
- [ ] 構文確認、テスト、ビルド、起動確認をプロジェクトに応じて実施する。
- [ ] アプリがある場合は主要操作とブラウザコンソールエラーの有無を確認する。
- [ ] 初回コミット前に、変更内容、影響範囲、未確認事項を整理する。
- [ ] ユーザーの明示的な許可を得て初回コミットを作成する。
- [ ] ユーザーの明示的な許可を得てGitHubへ初回pushする。
- [ ] ローカルブランチとGitHubの対象ブランチが同期していることを確認する。

### 7. 継続開発確認

- [ ] 新しい環境または新しいAIが文書だけでセットアップと現状把握を行えることを確認する。
- [ ] 次の開発課題、優先順位、完了条件、未確認事項が明確であることを確認する。
- [ ] テスト、ビルド、レビュー、Git運用の方法が再現可能であることを確認する。
- [ ] 重要判断と作業結果を `development-management` へ継続して反映する運用を確認する。

## 「プロジェクト化完了」の定義

次の条件をすべて満たした状態を「プロジェクト化完了」と定義します。

- [ ] GitHub管理を開始している。
- [ ] 正式ソースが確定している。
- [ ] READMEが整備されている。
- [ ] `docs/` が整備されている。
- [ ] AI引き継ぎ文書が整備されている。
- [ ] `development-management` に登録されている。
- [ ] ローカルとGitHubのGit同期が完了している。
- [ ] 新しいAIまたは開発者が継続開発できる。

一部が未確認、未同期、または暫定状態の場合は「プロジェクト化進行中」とし、完了扱いにしません。
