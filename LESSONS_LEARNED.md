# 教訓

ここには再発防止のための知見のみを記録する。
不具合だけでなく、運用改善・設計改善・AIとの開発で得られた教訓も記録対象とする。

- `robocopy /E` ではコピー先に古い `_internal` が残る。ランタイム領域は完全同期が必要。
- Python 3.12と3.13のランタイムが共有版に混在すると、開発環境では動いても共有版だけ起動に失敗することがある。
- 正式ソース以外で開発すると、変更・Git履歴・進行状況が分散し、どれが正しいか判断できなくなる。
- 未コミット変更はGitHubから確認できない。引き継ぎ時はローカルのGit状態を必ず確認する。
- Windows PowerShell 5.1で実行するスクリプトは、互換構文と文字コードを意識する。文字化けや解析エラーを起動前に確認する。
- 実行時データとソース、配布物を分離する。共有版更新で業務データを削除・上書きしない。
- `master_settings.json` がない環境ではビルドが失敗し得る。実値をGit管理せず、ビルド時の扱いとサンプル設定を明確にする。
- 「最新タグ」「未コミットの最新状態」「実運用中の版」は別物として記録する。
- Codexのサンドボックスでは正式ソース外への書き込みや一部Git操作が制限される。権限エラー時に別フォルダへ回避コピーせず、対象と権限を確認する。
